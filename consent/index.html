SMS Consent Process

How Consent is Collected

WellCheck collects explicit prior consent for SMS alerts through a multi-step verification process:

  1. User adds contact: A WellCheck user manually adds a trusted contact within the app, providing their name, phone number, and email address
  2. Email consent request sent: The contact receives an email explaining the service and requesting consent
  3. Contact reviews consent form: Contact clicks a link to review a detailed consent form with explicit SMS messaging agreement
  4. Explicit SMS acceptance required: Contact must click "I Accept and Agree to Receive SMS Messages" button to consent
  5. Welcome SMS sent: After accepting, contact receives a welcome SMS with opt-out instructions
Important: Contacts must explicitly accept via the consent form before receiving any SMS alerts. The consent form prominently displays: "I agree to receive SMS text messages from WellCheck" with detailed information about message types, frequency, and opt-out options.

Opt-Out Process

Recipients can opt out at any time by:
  • Declining the email consent request (before any SMS sent)
  • Replying STOP to any SMS message
  • Replying UNSUBSCRIBE
  • Replying HELP for more information

Opt-outs are processed immediately and permanently. Once opted out, the contact will not receive any further messages from WellCheck.

Message Frequency

Contacts receive SMS messages only when:

  • After accepting SMS consent: One welcome message with setup instructions
  • During emergencies: When the WellCheck user misses a scheduled health check-in (rare)

Expected frequency: Very low - typically zero messages per month unless there's a genuine emergency.

Data Collected for Consent

To manage consent, we store:

  • Contact email address (for consent request)
  • Contact phone number (displayed on consent form for verification)
  • Consent status (pending/confirmed/rejected)
  • Consent timestamp
  • User's full name (displayed in consent request)

Data retention: Consent records stored for 7 days if no response (then expired), permanently if accepted or rejected.

Consent Expiration

Email consent requests expire after 7 days if not responded to. The contact can be re-added and will receive a new consent request.

TCPA Compliance

WellCheck's SMS consent process is designed to comply with the Telephone Consumer Protection Act (TCPA):

  • ✅ Explicit prior consent - Clear "I agree to receive SMS text messages" language
  • ✅ Separate from other agreements - SMS consent is prominently displayed in its own section
  • ✅ Clear disclosure - Message types, frequency, and rates disclosed upfront
  • ✅ Easy opt-out - Reply STOP at any time
  • ✅ No auto-consent - Contacts must actively click Accept button
  • ✅ Documented process - All consent actions timestamped and stored
