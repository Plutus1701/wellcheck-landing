SMS Consent Process

How Consent is Collected

WellCheck collects explicit prior consent for SMS alerts through a multi-step verification process:

The WellCheck user is solely responsible for initiating and managing the consent process for their emergency contacts. WellCheckApp LLC provides the consent mechanism but bears no responsibility for ensuring individual users complete the process correctly.

  1. User adds contact: A WellCheck user manually adds a trusted contact within the app, providing their name and phone number
  2. User texts consent link: The app prompts the user to send a pre-written text message from their own phone — the message includes a personal explanation and a unique consent link
  3. Contact reviews consent form: Contact taps the link to review a detailed consent form with explicit SMS messaging agreement
  4. Explicit SMS acceptance required: Contact must click "I Accept and Agree to Receive SMS Messages" button to consent
  5. Welcome SMS sent: After accepting, contact receives a welcome SMS with opt-out instructions
Important: Contacts must explicitly accept via the consent form before receiving any SMS alerts. The consent form prominently displays: "I agree to receive SMS text messages from WellCheck" with detailed information about message types, frequency, and opt-out options.

Consent Link via Text

When you add a contact, WellCheck generates a unique consent link and opens your native Messages app with a pre-written text ready to send. The message comes from your own phone number — so your contact recognizes it's from you — and includes a clear explanation of what WellCheck is and a link to the consent form.

Key Consent Elements

  • Prominent SMS consent section with green border highlighting
  • Explicit statement: "I agree to receive SMS text messages from WellCheck"
  • Phone number displayed for verification
  • List of message types:
    • One welcome message after accepting
    • Emergency alert messages if user misses check-in
    • Expected frequency: Very rare (only during emergencies)
  • "Message and data rates may apply" disclaimer
  • "Reply STOP to opt out" instructions
  • Clear action buttons: Accept or Decline

Opt-Out Process

Recipients can opt out at any time by:
  • Declining the consent request (before any SMS sent)
  • Replying STOP to any SMS message
  • Replying UNSUBSCRIBE
  • Replying HELP for more information

Opt-outs are processed immediately and permanently. Once opted out, the contact will not receive any further messages from WellCheck.

Message Frequency

Contacts receive SMS messages only when:

  • After accepting SMS consent: One welcome message with setup instructions
  • During emergencies: When the WellCheck user misses a scheduled health check-in (rare)

Expected frequency: Very low - typically zero messages per month unless there's a genuine emergency.

Data Collected for Consent

While a consent request is pending, we store:

  • Contact name (to personalize the consent form)
  • Contact phone number (displayed on the consent form for verification)
  • User's full name (displayed in the consent request)
  • Consent status and request timestamp

Once resolved (accepted, declined, or expired), we immediately delete the email address and names. Only the following is retained for TCPA compliance:

  • Phone number
  • Consent status (confirmed/rejected/expired)
  • Timestamps (requested, responded)

Pending requests auto-expire after 7 days if no response, at which point PII is also deleted.

Consent Expiration

Consent requests expire after 7 days if not responded to. The contact can be re-added and you can send a new consent link at any time.

TCPA Compliance

WellCheck's SMS consent process is designed to comply with the Telephone Consumer Protection Act (TCPA):

  • ✅ Explicit prior consent - Clear "I agree to receive SMS text messages" language
  • ✅ Separate from other agreements - SMS consent is prominently displayed in its own section
  • ✅ Clear disclosure - Message types, frequency, and rates disclosed upfront
  • ✅ Easy opt-out - Reply STOP at any time
  • ✅ No auto-consent - Contacts must actively click Accept button
  • ✅ Documented process - All consent actions timestamped and stored
