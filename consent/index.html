WellCheck SMS Consent Process

How Consent is Collected

WellCheck collects explicit consent for SMS alerts through the following process:

  1. User Action: A WellCheck user manually adds a trusted contact within the app
  2. Intro Message Sent: The contact receives an automated introduction message explaining the service
  3. Explicit Opt-In: Contact can reply STOP to immediately opt out, or continue receiving alerts by taking no action

Sample Intro Message (Sent When Contact Added)

Hi [CONTACT NAME]! [USER FULL NAME] added you as an emergency contact in WellCheck, a safety monitoring app. You'll receive automated alerts ONLY if [USER FULL NAME] misses a health check-in, which may indicate a medical emergency. Expected frequency: Rarely (hopefully never) Learn more: wellcheckapp.com Reply STOP to opt out | HELP for info

Sample Emergency Alert

⚠️ WELLCHECK EMERGENCY ALERT [USER FULL NAME] missed a safety check-in and may need immediate assistance. Scheduled check-in: [TIME] on [DATE] Grace period ended: [TIME] Alert sent: [CURRENT TIME] Please verify [USER FULL NAME] is safe. Call immediately: [USER PHONE NUMBER] Reply STOP to opt out

Opt-Out Process

Recipients can opt out at any time by:
  • Replying STOP to any message
  • Replying UNSUBSCRIBE
  • Replying HELP for more information

Opt-outs are processed immediately and permanently.

Message Frequency

Contacts receive messages only when:

  • They are first added (intro message - once)
  • The user misses a check-in (emergency alert - rare)

Expected frequency: Very low - emergency alerts are sent only during genuine emergencies.
